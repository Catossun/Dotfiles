# 工作待辦管理

維護以週為單位的待辦系統。每週一個檔案，索引追蹤每週是否完成；非排程想法放 backlog。

## 檔案結構

todo/
├── INDEX.md          ← 週清單 + 完成狀態（永遠載入，保持輕量）
├── backlog.md        ← 不綁週的想法/長期點子
└── weeks/
    └── YYYY-Www.md   ← 每週一檔（ISO 週數）

## 何時讀哪個檔案（節省 token）

- 「列出待辦 / 我現在在做什麼 / 還有什麼要做」→ 讀 INDEX.md + 當週檔案
- 「加入待辦」→ 讀當週檔案 + 編輯
- 「丟進 backlog」→ 只讀 backlog.md
- 「上週做了什麼 / 翻舊紀錄」→ 從 INDEX.md 拿到對應週檔再讀
- **永遠不要一次把所有週檔都讀進來**

## 取得「本週」是哪一週

ISO 週的計算：
```bash
date +"%G-W%V"            # bash（git bash 也行）
date -d "2026-05-08" +"%G-W%V"   # 指定日期

PowerShell 5.1 沒有 `System.Globalization.ISOWeek`，所以用 bash 算。週一是一週的第一天，週日是最後一天。

## 待辦項目格式

- [ ] **標題** `[優先級]` — 一行簡述
  - 建立：YYYY-MM-DD
  - 相關：[TICKET-123](https://...) 或 `path/to/file.ts:42`
  - 備註：(選填)

| 欄位 | 規則 |
|---|---|
| [ ] / [x] | 未完成 / 已完成 |
| 優先級 | [P0] 緊急、`[P1] 重要、[P2] 一般、[P3]` 有空再做 |
| 日期 | 永遠用絕對日期 YYYY-MM-DD**，禁止寫「今天」「明天」 |

## 週檔內部分區

每個 `weeks/YYYY-Www.md` 用三段：

1. 🔥 進行中** — 這週實際在動的，理想 ≤ 3 項
2. 📋 待處理 — 已排這週做、按優先級排序
3. ✅ 已完成 — 完成的放最上方，附完成日期

未做完轉到下週的不留在這裡（見「結束本週」）。

## 標準操作流程

### 新增待辦到本週
1. Read INDEX.md 拿到當週檔名（或用 date +%G-W%V 算）
2. Read 當週檔
3. Edit：附加到 📋 待處理（依優先級插正確位置）
4. 沒指定優先級就先設 [P2] 並回報「我先設 P2，需要調整嗎？」

### 開始處理某項
- 從 📋 待處理 移到 🔥 進行中（同一個週檔內）

### 完成項目
1. [ ] → [x]
2. 移到 ✅ 已完成 最上方
3. 描述後追加「完成：YYYY-MM-DD」

### 取消／放棄
- 移到 ✅ 已完成，標題前綴 `取消：`，補原因

### 加入 backlog（非排程）
- 直接 Edit backlog.md，加一條「- 標題 — 描述」即可，不需要日期或優先級

### 從 backlog 排程到某週
- 從 backlog.md 剪掉
- 貼到目標週的 📋 待處理，補上建立日期與優先級

### 結束本週（標記為已完成）
觸發：使用者說「這週的標完成」「W19 收尾」「開新週」之類。

1. Read 當週檔，檢查 🔥 進行中 + 📋 待處理 是否還有未完成項
2. 若有未完成**：
   - 詢問使用者：「W19 還有 N 項沒做完：[列出標題]。要轉到 W20 嗎？還是放 backlog / 取消？」
   - 依回答處理（轉週 = 在新週檔加入這些項目，原項目改 `[x]` 並加註「↺ 轉至 Wxx」放到 ✅ 已完成）
3. 在 INDEX.md 把該週從「進行中」搬到「已完成」並改成 `- [x]`

### 開新週
1. 用 bash `date +%G-W%V` 算當前 ISO 週
2. 算當週的週一/週日日期（用 `date -d "..." +%m-%d`）
3. 建立 `weeks/YYYY-Www.md`（用範本，見下）
4. 在 INDEX.md「進行中」段加一行
5. 若上週還沒結束，先按「結束本週」流程詢問處理

## 顯示／查詢

- 「列出待辦 / 還有什麼要做」→ 預設只列當週的 🔥 進行中 + 📋 待處理，用條列回覆，不要把整份檔案貼出來
- backlog 只在使用者明確問才列
- 歷史週只在使用者點名才讀

## 與 Jira 整合（選用）

使用者環境有 atlassian MCP：
- **從 Jira 同步**：`mcp__atlassian__getJiraIssue` 取 ticket，建立對應待辦，「相關」欄填 Jira URL
- **狀態回寫 Jira**：使用者明確要求才做（`mcp__atlassian__transitionJiraIssue`），不要自動同步
- 不要主動爬 Jira 來建立待辦，使用者點名才做

## 維護原則

- **編輯前一定先 Read**（檔案可能被使用者手改過）
- 每次編輯 INDEX.md 或週檔，記得改最上方「最後更新」日期
- 操作後簡短回覆「已加到 W19 待處理 / 已標記完成」即可，不要把整份檔案輸出
- 日期一律用系統 currentDate 的絕對值，不要猜

## 週檔範本

```markdown
# YYYY-Www (M/d–M/d)

> 狀態：進行中

## 🔥 進行中

_(目前沒有項目)_

## 📋 待處理

_(目前沒有項目)_

## ✅ 已完成

_(目前沒有項目)_
```

## 範例：新增一筆待辦

**使用者**：「加一個待辦，要把 ABC 服務的 log 改用結構化格式，這個有點急」

**動作**：
1. Read INDEX.md → 知道當週是 2026-W19
2. Read weeks/2026-W19.md
3. Edit：在 📋 待處理 加：
   ```markdown
   - [ ] **ABC 服務改結構化 log [P1] — 把現有 plain text log 改 JSON 結構化
     - 建立：2026-05-08
   ```
4. 改檔案最上方日期戳記
5. 回覆：「已加到 W19 待處理（P1）。要關聯 Jira ticket 嗎？」
